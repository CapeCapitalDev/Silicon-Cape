## OUTPUT A — POWERPOINT PRESENTATION

### When to use
- Presenting a theme to CAPE Capital (internal weekly)
- Pitching an investment idea
- Briefing colleagues or clients
- Structuring a discussion around a portfolio or macro view

### Audience note
CAPE weekly audience has varied levels of financial knowledge. Always frame the theme before diving into the argument. Do not assume prior familiarity with the subject.

### Cowork workflow

**Step 1 — Generate Perplexity visual sourcing prompt**
At the start of every presentation task, generate the visual sourcing prompt below (filled in for the specific theme). William runs it in Perplexity Deep Research and returns the output containing image URLs and attribution lines.

**Step 2 — Build the deck**
Once William returns with the Perplexity visual output, build the full PPTX directly in Cowork using the CAPE format and slide structure below. Embed image URLs and attribution lines as specified. No add-in required.

---

### Perplexity visual sourcing prompt template

```
PERPLEXITY VISUAL SOURCING PROMPT — [THEME NAME]
Reference date: [today's date]

Find the 20 best available visuals (charts, tables, maps, graphs) related
to this theme. Recency target: within the last 12 months where possible.

For each visual return:
- The image, chart, or table (screenshot, jpg, or image format)
- Direct URL to the image or page containing it
- File format if determinable (jpg, png, pdf page, embedded chart)
- Source name (e.g. IMF, Goldman Sachs, Bloomberg, IEA, McKinsey)
- Publication date
- One-sentence description of what the visual shows
- Attribution line for slide footer:
  Source: [Organisation] · [Publication Title] · [Month Year]

Source quality hierarchy (in order of preference):
Tier 1: IMF, World Bank, BIS, central banks, IEA, OECD
Tier 2: Goldman Sachs, Morgan Stanley, McKinsey, BCG, Bloomberg Intelligence
Tier 3: CB Insights, PitchBook, Statista, ARK Invest, Refinitiv/LSEG
Tier 4: FT, WSJ, Reuters, Bloomberg News
Tier 5: social media, blogs, unattributed aggregators — avoid where possible

Return all 20 results with full attribution before summarising anything.
```

---

### Slide structure

Adapt the number of slides to the material. The structure below is the required sequence; not all sections require a dedicated slide if the content is thin.

1. **Cover** — theme name, key message as subtitle, date, presenter
2. **Context / Why Now** — frame the theme for a mixed-knowledge audience; one or two sentences on why this matters today
3. **Core Thesis** — the central argument in plain language; what the market may be missing
4. **System Map or Value Chain** — sector structure, key actors, margin pools, chokepoints
5. **Evidence and Market Logic** — demand drivers, supply dynamics, pricing, adoption, policy
6. **Investable Implications** — listed beneficiaries, indirect exposure, what to avoid, timing
7. **Risks and Debate** — key risks ranked; strongest alternative interpretation
8. **Conclusion and Highlight** — end on a forward-looking point worth remembering; never leave the audience on a low note
9. **Appendix / Sources** — full source list if needed

Agenda slide: include only if the deck is long enough to need navigation (more than 8 content slides).

---

### CAPE format specification
- Palette: dark navy (#0A1628) background, gold (#C9A84C) accents, white text
- Font: Arial throughout; minimum 18pt body text
- One core idea per slide
- Headlines carry the argument — they are not labels
- No walls of text; bullets maximum 4 per slide
- Charts and visuals clarify the point; they do not decorate
- Attribution line in footer for every sourced visual:
  Source: [Organisation] · [Publication Title] · [Month Year]

---

### Quality bar
- The deck should have narrative flow from first to last slide
- It should be speakable — written for a presenter, not a reader
- The audience should not have to decode the point themselves
- The conclusion slide must leave the room with something worth remembering

## OUTPUT B — 1-PAGER

### When to use
- After a presentation has been completed (derives from the deck)
- As a standalone summary for internal circulation
- As a filing note for the CAPE thematic research folder

### Format specification

Derived from William's established format (e.g. "Iran explained", "Proteins", "Stablecoins").

**Structure:**

```
[THEME NAME] — Summary

[Title] -- Presentation has been added to the usual folder (in pptx and in pdf):
[SharePoint link placeholder]
This one pager can be found [SharePoint link placeholder]

• [Bold label]: [Concise, data-rich sentence. Real numbers. Sourced facts.]
• [Bold label]: [Concise, data-rich sentence.]
• [Bold label]: [Concise, data-rich sentence.]
[... 8–12 bullets total ...]
• [Final bullet — forward-looking or upside scenario, set apart visually]:
  [The highlight. End on a note worth remembering — an opportunity,
   a scenario, or a consequence that reframes the whole picture.]
```

**Rules:**
- Bold label + colon + data-rich content. Not general description — actual figures, named actors, specific mechanisms.
- 8–12 bullets. No more.
- No paragraphs or long narrative text anywhere.
- Final bullet is always forward-looking or scenario-based. It is the note the reader carries away.
- Total length: approximately half a page in Word; 200–250 words maximum.
- Language: concise, professional, readable by an intelligent non-specialist.
- File format: .docx (primary) and .md (portable version).

### Prompt pattern (when deriving from a presentation)

```
Read the attached [THEME] presentation.

Produce a 1-pager in William's investment note format:
- Header: "[THEME] — Summary"
- Filing line referencing the usual CAPE folder
- 8–12 bullet points using the format: **Bold label**: [data-rich sentence]
- Each bullet must contain real figures, named actors, or specific mechanisms
  — not general description
- Final bullet: forward-looking or upside scenario, set apart as the highlight
- No paragraphs, no long narrative, no commentary outside the bullets
- Total: 200–250 words maximum
- Output as .docx file

Do not summarise the slides — extract the investment-relevant facts.
```
